# Repository Guidelines

## Project Structure & Module Organization
This is a React + TypeScript app built with Vite. Key locations:
- `src/` holds application code. Entry points are `src/main.tsx` and `src/App.tsx`.
- `src/components/` contains reusable UI components.
- `src/styles/` and `src/index.css` contain global and feature styles.
- `src/assets/` stores static assets (for example, `ascii.txt`).
- `src/types/` defines shared TypeScript types.
- `public/` hosts static files copied as-is at build time.
- `dist/` is the build output generated by Vite (do not edit manually).
- `links.txt` is a content source used by the UI.
- `commands.txt` maps terminal commands and aliases (for example, `projects` and `project`).

## Build, Test, and Development Commands
Run commands from `my-app/`:
- `npm run dev` starts the Vite dev server with HMR.
- `npm run build` type-checks with `tsc -b` and produces a production build.
- `npm run preview` serves the production build locally.
- `npm run lint` runs ESLint across the codebase.
- `npm run links` prints `links.txt` (useful for validating link content).

## Coding Style & Naming Conventions
- Use 2-space indentation in TS/TSX/CSS to match existing files.
- Prefer named React components (PascalCase) and hooks (camelCase with `use` prefix).
- Keep file names in PascalCase for components (for example, `TerminalHeader.tsx`).
- Follow ESLint recommendations in `eslint.config.js`; fix lint errors before PRs.

## Testing Guidelines
There is no test framework configured yet. If adding tests:
- Keep test files next to the code they cover or under a `tests/` folder.
- Use `.test.ts`/`.test.tsx` naming to align with common tooling.
- Document the new test command in this file.

## Commit & Pull Request Guidelines
- Commit messages follow a conventional pattern like `feat: ...` (see `git log`).
- PRs should include a short description, key behavior changes, and screenshots for UI updates.
- Link relevant issues or tasks when available.

## Configuration Tips
- TypeScript configs live in `tsconfig.app.json` and `tsconfig.node.json`.
- The Vite config is `vite.config.ts`; update it when adding plugins or aliases.
